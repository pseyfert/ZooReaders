37 through the examples
38 ====================
39 
40 When reading through the examples _please_ don't start just somewhere. Go
41 through them in the following order.
42 
43 1. [hello_options.cc] the most readable intro into options I could think of.
44 2. [hello_logstreams.cc] just putting some noise to the logstreams. play around
45  with the -v option. Shows how functions can be called when an option gets
46  parsed, both using the value as well as not using it (mind the template of option::myfun).
47 3. [hello_logstreams_glob.cc] not to introduce code duplication, most of
48  ```hello_logstreams.cc``` is exported to the helpers library. This shows how
49  you're supposed to use options.
50 4. [ZooReader_basic.cc] a minimal executalbe close to the old ZooReader
51  example. NB: It shows a check if the branch exists and tells the user all
52  available branch names.
53 5. [ZooReader_options.cc] adding options as in hello_options.cc to the
54  ZooReader.
55 6. [ZooReader_glob_options.cc] putting things together from
56  hello_logstreams_glob.cc and ZooReader_options.cc

58 the ZooGui
59 ==========
60 
61 This is just a toy project. Two ways of using it:
62 
63 Firstly, in interactive root:
64 ```
65 root -l
66 gSystem->Load("Zoo/libZooROOT.so")
67 gSystem->Load("ZooFunctors/libZooFunctorROOT.so")
68 .L ZooGui.cpp+O
69 ZooGui gui
70 gui.LoadFile("/path/to/your/ntuple.root")
71 ```
72 
73 Secondly executed:
74 ```
75 make ZooGui
76 ./ZooGui </path/to/your/ntuple.root>
77 ```
78 
79 Thirdly, due to limitations of CINT you cannot actually develope your plot on
80 runtime. This could be much more awesome with the PyROOT methods shipped with
81 Phys/ZooWriter/examples. I'm curious if this gets fantastic with cling/clang.
