Detailed Description

class to handle logging to an ostream

the idea behind this class is that logging to some kind of ostream should be convenient:

  • it should be possible to switch logging on and off
  • the log file should contain which part of the software is logging
  • should behave more or less like cout/cerr from a user's point of view

get a logstream

get and release

get a message stream based on ostream s, with a message level mylevel and a given name the reference to the logstream object can be used very much like a C++ ostream object: 

            logstream& mylog =
                    logstream::get(std::cout, 2, "MyLog");
            mylog << "Hello, world!" << std::endl;

should produce something similar to 

            MyLog   Hello, world!

on the standard output, provided that the global log level is no higher than 2 (see below), i.e. a logstream will only produce output when its log level is greater or equal the global log level

don't use get excessively, rather keep your reference to the logstream until you don't need it any longer, because the get operation is not cheap there are some pre-canned stream available for general use, see below

make a log stream permanent

make a log stream permanent, i.e. set a flag such that release will become a no-op for that stream this is useful to make sure that standard streams remain available

usage example: 

        logstream& logic =
            logstream::get(std::cerr, 7, "LOGIC").makePermanent();
